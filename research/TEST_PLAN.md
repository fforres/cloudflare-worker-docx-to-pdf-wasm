# Test plan — shared across all three approaches

20–25 real DOCX files under `fixtures/`, split into three tiers. Reference PDFs are generated once by LibreOffice headless and committed under `reference-pdfs/` with the same relative path and `.pdf` extension.

## Reference PDFs

Generated by:
```bash
soffice --headless --convert-to pdf --outdir reference-pdfs/<tier>/ fixtures/<tier>/*.docx
```

These PDFs are treated as **ground truth**. They are not Word's output; LibreOffice's pagination differs from Word's, but for content-level comparison this is good enough and reproducible.

## Per-document scorecard

`harness/` produces, for each (approach, document) pair:

| Metric | How measured | Critical? |
|---|---|---|
| **Conversion succeeded** | Exit code 0 + output PDF non-empty | Yes |
| **Text token recall** | `pdftotext` both PDFs, tokenize, `intersection / |reference_tokens|`. | Yes (target ≥ 0.85) |
| **Page count delta** | `pdfinfo` Pages field. Report `|gen - ref| / ref`. | Important (target ≤ 0.2) |
| **Image count delta** | `pdfimages -list` row count. | Important |
| **Generated PDF size (KB)** | `wc -c` | Informational |
| **Conversion time (ms)** | native binary, `time` | Informational |
| **First-page SSIM** | `pdftoppm -r 100 -f 1 -l 1`, then `magick compare -metric SSIM`. Skipped if no ImageMagick. | Nice-to-have |

## Aggregate scorecard per approach

```
| Tier | Conv-OK | Text recall (avg) | Page Δ (avg) | Image Δ (avg) | Size (avg) |
|------|---------|-------------------|--------------|---------------|------------|
| T1   |  5 / 5  |        0.94       |     0.05     |     0.00      |    24 KB   |
| T2   |  8 / 10 |        0.81       |     0.18     |     0.40      |    62 KB   |
| T3   |  4 / 10 |        0.55       |     0.32     |     0.60      |   140 KB   |
```

Plus a "static" row per approach:
- Compressed WASM size (`wasm-opt -Oz` then `gzip -9 -c | wc -c`).
- Native build time.
- Pages of OOXML spec the approach claims to cover (from its README, anecdotal).

## Pass/fail gates

For an approach to advance to the CF Worker step:
1. **Tier 1**: 5/5 conversions succeed AND average text recall ≥ 0.85.
2. **Compressed WASM** ≤ 10 MiB (the hard CF Workers limit).
3. **Tier 2**: at least 6/10 conversions succeed.
4. **Tier 3**: best-effort, no gate.

An approach that passes (1) + (2) but fails (3) is still a candidate for Workers — we can downgrade ambition. An approach that fails (2) is a Containers candidate, not Workers, and is parked unless all three fail (2).

## Harness invocation

```bash
# from repo root
./harness/score.sh approach-a-custom
./harness/score.sh approach-b-typst
./harness/score.sh approach-c-rdocx
```

Outputs:
- `results/<approach>/<tier>/<filename>.json` per document
- `results/<approach>/summary.md` aggregate
- `results/comparison.md` cross-approach (after all three run)

## Conventions for the per-approach CLIs

Each approach exposes the same native CLI shape so the harness can drive them uniformly:

```
./target/release/<approach>-cli <input.docx> <output.pdf>
```

Exit 0 on success, non-zero on failure. Print nothing on stdout except errors. The harness times it with `/usr/bin/time -l`.
